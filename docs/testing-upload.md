# Authenticated Upload Test Checklist

Use this checklist against a real Supabase project with `NEXT_PUBLIC_REQUIRE_AUTH=true`.

1. Open `/login`.
2. Sign up or sign in with a real Supabase-authenticated user.
3. Open `/dev/diagnostics` locally and confirm:
   - Auth session present: `true`
   - Profile found: `true`
   - Workspace id is populated
   - Storage bucket reachable: `true`
4. Open `/upload`.
5. Upload a real PDF.
6. In Supabase Storage, confirm a file exists in the private `documents` bucket under:
   - `{workspace_id}/documents/...`
7. In Supabase Table Editor, confirm rows were created:
   - `documents`
   - `uploads`
   - `document_versions`
   - `processing_jobs`
8. Call `/api/jobs/process` while authenticated.
9. Confirm processing rows move through:
   - `queued`
   - `running`
   - `completed`
10. Confirm OCR/extraction output rows exist:
    - `ocr_results`
    - `extraction_results`
11. Open `/documents`.
12. Click the uploaded document and confirm `/documents/[id]` opens.
13. Search for OCR text from the document using global search.
14. Open `/convert`.
15. Select the uploaded document and start a conversion.
16. Call `/api/jobs/process` again.
17. Confirm the `conversions` row is `completed` and has `download_path`.
18. Click Download and confirm the converted output downloads from Storage.

Expected fallback behavior:

- If no real OCR/extraction/conversion provider is configured, results are generated by internal mock providers.
- Mock providers should still write `ocr_results`, `extraction_results`, `processing_jobs`, audit logs, and conversion metadata.
